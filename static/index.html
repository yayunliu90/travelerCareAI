Fills the symptom text, where you are, where you are from, map search, and sometimes language / query strategy. Then click Get guidance.

Uses GOOGLE_MAPS_SERVER_KEY and SERPER_API_KEY on the server. The research step is steered toward official local government / public-health pages for the trip location when possible. Pin a location on the map (or set “Where you are”) so Places can run. Browser Maps key alone is not enough for server Places.

Used for localized guidance. If empty, the server may use DEFAULT_TRAVEL_LOCATION from .env, or stay location-agnostic. The map search below can also fill this when you pick a place.

Helps compare how healthcare usually works where you are now versus what you may be used to at home (access, referrals, payment, pharmacies, emergency norms). If empty, the server may use DEFAULT_HOME_COUNTRY from .env. Important: fill this field (or rely on that env default) — writing “I’m from …” only inside the symptom box does not send traveler_home_country to the model, so it may still ask where you are from.

This demo combines rule-based checks, simple retrieval over a tiny local corpus, and optional OpenAI for wording — it is not medical advice.
